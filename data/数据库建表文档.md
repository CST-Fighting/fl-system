# 联邦学习算力网络平台 — 数据库建表文档

> 数据库类型: **PostgreSQL**
> 目标库: **computing_network**
> 管理员: **cn_admin**
> 连接: `postgresql://cn_admin:slwl@10.129.209.249:5433/computing_network`

---

## 表总览

| 序号 | 表名 | 中文名 | 说明 |
|:---:|------|--------|------|
| 1 | `supercomputing_centers` | 超算中心资源表 | 存储国家级/区域超算中心的基本配置与实时状态 |
| 2 | `node_monitor` | 算力节点监控表 | 算力节点的时序监控数据（CPU/GPU/内存/网络） |
| 3 | `edge_nodes` | 边缘节点表 | 边缘计算节点的硬件配置、状态与健康信息 |
| 4 | `tasks` | 任务主表 | 所有计算任务的核心信息（请求、分配、状态） |
| 5 | `federated_tasks` | 联邦学习扩展表 | 联邦学习任务的聚合算法、轮次等扩展字段 |
| 6 | `task_participants` | 任务参与者表 | 联邦学习中各客户端的贡献与收益记录 |

**表关系:**

```
tasks (1) ──── (1) federated_tasks        -- 一对一扩展
tasks (1) ──── (N) task_participants      -- 一对多参与记录
node_monitor (N) ── node_id ── 边缘/算力节点  -- 多对一关联
```

## 一、supercomputing_centers（超算中心资源表）

存储超算中心的基本信息和运行状态，数据来源于 `supercomputing data.json`。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | SERIAL | PRIMARY KEY | 自增主键 |
| `name` | VARCHAR(100) | NOT NULL | 超算中心名称，如"天河二号" |
| `longitude` | DOUBLE PRECISION | 可空 | 经度坐标 |
| `latitude` | DOUBLE PRECISION | 可空 | 纬度坐标 |
| `compute_power` | DOUBLE PRECISION | 可空 | 总算力，单位 PFLOPS（每秒千万亿次浮点运算） |
| `cpu_cores` | INT | 可空 | CPU 总核心数 |
| `gpu_type` | VARCHAR(50) | 可空 | GPU 型号，如"NVIDIA A100" |
| `gpu_count` | INT | 可空 | GPU 总数量 |
| `network_bw` | VARCHAR(20) | 可空 | 网络带宽描述，如"100Gbps" |
| `capability_score` | DOUBLE PRECISION | 可空 | 综合能力评分 (0-100) |

---

## 二、node_monitor（算力节点实时监控表）

时序数据表，每条记录代表某节点在某一时刻的监控快照。数据来源于 `server_monitor_node01.csv` / `node02.csv`。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | SERIAL | PRIMARY KEY | 自增主键 |
| `node_id` | VARCHAR(50) | NOT NULL | 节点标识，对应 `node_01` / `node_02` |
| `timestamp` | TIMESTAMPTZ | NOT NULL, DEFAULT NOW() | 数据采集时间（带时区） |
| `cpu_usage` | DOUBLE PRECISION | 可空 | CPU 使用率，百分比 0-100 |
| `memory_usage` | DOUBLE PRECISION | 可空 | 内存使用率，百分比 0-100 |
| `disk_usage` | DOUBLE PRECISION | 可空 | 磁盘使用率，百分比 0-100 |
| `gpu_usage` | DOUBLE PRECISION | 可空 | GPU 平均使用率，百分比 0-100 |
| `network_rx` | BIGINT | 可空 | 网卡接收速率 (KB/s) |
| `network_tx` | BIGINT | 可空 | 网卡发送速率 (KB/s) |

**索引:** `idx_node_monitor_time` — 组合索引 `(node_id, timestamp DESC)`，加速按节点查询最近监控数据。

**数据特点:**
- 高频写入表，建议定期归档历史数据
- 当前数据源每秒采集一次，4 个 GPU 各有独立利用率/显存数据

## 三、edge_nodes（边缘节点表）

边缘计算节点的静态配置与动态状态，数据来源于 `edge_nodes.csv`。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | SERIAL | PRIMARY KEY | 自增主键 |
| `node_name` | VARCHAR(100) | NOT NULL | 节点名称，如 `edge-shanghai-01` |
| `status` | VARCHAR(20) | DEFAULT 'online' | 节点状态：`online` / `offline` / `maintenance` |
| `compute_type` | VARCHAR(20) | 可空 | 计算类型：`CPU` / `GPU` / `GPU+CPU` |
| `cpu_usage` | DOUBLE PRECISION | 可空 | CPU 使用率 (%) |
| `memory_usage` | DOUBLE PRECISION | 可空 | 内存使用率 (%) |
| `gpu_usage` | DOUBLE PRECISION | 可空 | GPU 使用率 (%) |
| `disk_usage` | DOUBLE PRECISION | 可空 | 磁盘使用率 (%) |
| `network_bandwidth` | INT | 可空 | 网络带宽 (Mbps) |
| `health_score` | DOUBLE PRECISION | 可空 | 综合健康评分 (0-100) |
| `running_tasks` | INT | 可空 | 当前运行中的任务数 |
| `total_tasks_completed` | INT | DEFAULT 0 | 历史累计完成的任务数 |
| `registered_at` | TIMESTAMPTZ | 可空 | 节点首次注册时间 |
| `last_heartbeat` | TIMESTAMPTZ | 可空 | 最后一次心跳上报时间 |
| `updated_at` | TIMESTAMPTZ | DEFAULT NOW() | 记录最后更新时间 |

## 四、tasks（任务主表）

所有计算任务的核心信息，涵盖任务请求、资源分配和执行状态。数据来源于 `demo_fused_dataset.csv` 和 `tasks.csv`。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | VARCHAR(50) | PRIMARY KEY | 任务唯一 ID（来源于作业 ID） |
| `name` | VARCHAR(100) | 可空 | 任务名称，如 `Task-83810` |
| `user_id` | VARCHAR(100) | 可空 | 提交用户标识（已脱敏哈希值） |
| `group_id` | VARCHAR(100) | 可空 | 语义分组 ID，用于联邦学习客户端划分 |
| `type` | VARCHAR(20) | 可空 | 任务类型：`training`(训练) / `inference`(推理) / `aggregation`(聚合) |
| `status` | VARCHAR(20) | DEFAULT 'waiting' | 任务状态：`waiting` / `running` / `completed` / `failed` / `paused` |
| `priority` | VARCHAR(10) | DEFAULT 'medium' | 调度优先级：`low` / `medium` / `high` |
| `progress` | DOUBLE PRECISION | DEFAULT 0 | 执行进度百分比 (0-100) |
| `cpu_requested` | DOUBLE PRECISION | 可空 | 请求的 CPU 核数 |
| `gpu_requested` | DOUBLE PRECISION | 可空 | 请求的 GPU 卡数（支持小数表示 GPU 共享） |
| `mem_requested` | DOUBLE PRECISION | 可空 | 请求的内存容量 (GB) |
| `cpu_allocated` | DOUBLE PRECISION | 可空 | 调度器实际分配的 CPU 核数 |
| `gpu_allocated` | DOUBLE PRECISION | 可空 | 调度器实际分配的 GPU 卡数 |
| `mem_allocated` | DOUBLE PRECISION | 可空 | 调度器实际分配的内存 (GB) |
| `assigned_node` | VARCHAR(100) | 可空 | 调度器分配的目标节点 ID |
| `gpu_type` | VARCHAR(50) | 可空 | 任务要求的 GPU 类型：`T4` / `V100` / `A10` / `MISC` 等 |
| `duration` | INT | 可空 | 任务执行时长（秒） |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | 任务创建/提交时间 |
| `updated_at` | TIMESTAMPTZ | DEFAULT NOW() | 任务信息最后更新时间 |

**数据特点:**
- `requested` 字段为用户提交时声明的资源需求
- `allocated` 字段为调度器实际分配的资源量（可能少于请求）
- `duration` 对于已完成任务是实际执行时长，对于运行中任务是预估时长

---

## 五、federated_tasks（联邦学习任务扩展表）

联邦学习任务的扩展信息，与 `tasks` 表一对一关联。仅当任务类型为联邦训练时使用。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `task_id` | VARCHAR(50) | PRIMARY KEY, FK → tasks(id) | 关联的任务 ID |
| `aggregation_algo` | VARCHAR(100) | 可空 | 聚合算法：`FedAvg` / `FedProx` / `SCAFFOLD` / `FedNova` 等 |
| `encryption` | VARCHAR(100) | 可空 | 隐私保护方式：`同态加密` / `差分隐私` / `安全聚合` |
| `rounds_total` | INT | 可空 | 计划进行的总聚合轮次 |
| `rounds_current` | INT | DEFAULT 0 | 当前已完成的聚合轮次 |
| `global_model_version` | VARCHAR(50) | 可空 | 当前全局模型版本标识，如 `v1.3` |

---

## 六、task_participants（任务参与者表）

记录联邦学习任务中各客户端（参与者）的贡献统计和收益。与 `tasks` 表多对一关联。

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | SERIAL | PRIMARY KEY | 自增主键 |
| `task_id` | VARCHAR(50) | FK → tasks(id) | 关联的联邦任务 ID |
| `client_id` | VARCHAR(100) | NOT NULL | 客户端唯一标识（脱敏用户哈希） |
| `joined_at` | TIMESTAMPTZ | DEFAULT NOW() | 客户端加入该联邦任务的时间 |
| `samples_count` | INT | 可空 | 该客户端贡献的训练样本数量 |
| `compute_score` | DOUBLE PRECISION | 可空 | 算力贡献评分（综合 CPU/GPU 使用与时长） |
| `reward` | DOUBLE PRECISION | 可空 | 该客户端获得的激励收益 |
| `last_round` | INT | 可空 | 客户端最后参与的聚合轮次 |

**唯一约束:** `UNIQUE(task_id, client_id)` — 同一任务中同一客户端只能有一条记录。

## 数据导入对照

| 表名 | 数据源文件 | 导入方式 |
|------|-----------|---------|
| `supercomputing_centers` | `data/supercomputing data.json` | Python 脚本解析 JSON 后 INSERT |
| `node_monitor` | `data/server_monitor_node01.csv` / `node02.csv` | Python pandas 读取后批量 INSERT |
| `edge_nodes` | `data/edge_nodes.csv` | `psql \copy` 或 pandas 导入 |
| `tasks` | `data/tasks.csv` / `data/demo_fused_dataset.csv` | Python 脚本清洗转换后 INSERT |
| `federated_tasks` | 系统运行时动态生成 | API 端点写入 |
| `task_participants` | 系统运行时动态生成 | API 端点写入 |
