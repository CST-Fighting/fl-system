## 一、本周必须修的(自己一个人能干的)

### 1. 顶部 4 个统计卡数字别写死

"任务总数 23、运行中 12、空闲节点 6、资源利用率 68%" 这四个数是 stats 接口硬编码塞进去的,但 demands 接口实际只有 12 条任务。把卡片数字改成从 demands 真实算出来:
- 任务总数 = demands.length
- 运行中任务 = demands.filter(s => s.status === '运行中').length
- 空闲节点、资源利用率从真实节点表算

后端 stats 接口删掉硬编码的 23、12、6、68% 这一段。

### 2. "在线节点 99+" 那个徽标实际是 1397 个

antd Badge 默认 overflowCount 是 99,改成 9999 或者直接显示 1397:

```jsx
<Badge count={onlineCount} overflowCount={9999} />
```

### 3. 节点资源详情表前 5 行全 0%

前 5 行排在最前面的全是 0% / 0% / 0% / 0%,但表上面写着"平均 CPU 13.6% 平均 GPU 37.1%"。第一眼看像系统挂了。

把节点按 CPU/GPU 利用率倒序排,忙的放前面、闲的扔后面或折叠。也可以加个过滤器「只看活跃节点」。

### 4. 500 条 warning 节点全弹 notification 雪崩

进页面那一刻 500 条 warning 节点全用 `notification.error` 同时弹出来,叠满右半屏遮住下面所有内容。改成:
- 顶部红色横条只写一句 "503 节点严重告警,点击查看"
- 点开右侧 Drawer 列表分页查看(antd Drawer + Table virtual)
- 实在要弹也最多弹 Top 5,带"查看全部"按钮
- 循环弹 notification 那段代码删掉

### 5. 节点 ID 是 hash 全念不出来

现在显示成 `f26f3876c665a704d75a756f` 这种 24 位 hex。告警里也是这串。改成 `node-{城市}-{序号}` 格式,例如 `node-shenzhen-01`。

跟其他 3 个同学对齐命名规则(参见文末「四、跟其他人对齐」)。后端 mock 生成时加 `region` 字段,前端拼。

### 6. useEffect 在 StrictMode 双发

console 里 "===== 开始请求 TaskManagement 页面数据 =====" 打了两遍。加 AbortController 取消旧请求:

```js
useEffect(() => {
  const ctrl = new AbortController();
  fetch('/api/tasks/demands', { signal: ctrl.signal })...
  return () => ctrl.abort();
}, []);
```

### 7. antd 提示过时

- `Warning: [antd: Card] bordered is deprecated` → Card 改 `variant="outlined"`
- `Warning: [antd: notification] Static function can not consume context like dynamic theme` → 用 `App.useApp()` 拿到 notification 实例:
```js
const { notification } = App.useApp();
notification.error({ message: '...' });
```
别再 `import { notification } from 'antd'` 然后调静态方法。

## 二、立项书写过但还没做的(项目研究内容 2.1.1)

### 激励机制 + 服务定价

立项书 2.1.1 第三点:"研究基于激励机制与服务定价的强化联邦计算方法,设计算力节点参与资源调度的激励机制,利用服务定价机制量化计算任务的能耗及其数据的贡献价值"。

任务计价是任务管理的延伸,落在这边。要做:
- 任务的 CPU 核时 × CPU 单价 + GPU 核时 × GPU 单价 + 内存 GB·h × 内存单价 = 任务总价
- 节点的贡献金额(节点维度,跟张鼎功对接)
- 新增页面或子标签:"激励 / 定价"
- 单价配置(可调)+ 实时计算 + 排行榜

接口字段建议(后端):
```json
{
  "task_id": "...",
  "cost": { "cpu": 12.5, "gpu": 80.2, "memory": 4.3, "total": 97.0, "currency": "CNY" }
}
```

跟张鼎功对齐:他要在 8083 边缘控制层展示"边缘节点贡献金额排行榜",用的是你算出来的单价。两边接口字段对一下。

## 四、跟其他 3 个同学对齐(4 人本周开 30 分钟会)

这部分**不是一个人能干的**,4 个人本周必须开一次会确定下来,不然各干各的越走越远。

### 1. ID 命名统一

现在 4 个页面 5 套命名,评审一眼看出来不是同一个系统。统一成:

| 实体 | 命名规则 | 例子 |
|---|---|---|
| 算力节点(超算) | `node-{城市}-{序号}` | `node-shenzhen-01` |
| 边缘节点 | `edge-{城市}-{序号}` | `edge-chengdu-01` |
| 客户端 | `client-{4位序号}` | `client-0287` |
| 任务 | `task-{业务简称}-{序号}` | `task-fedtrain-99943` |

你这边把现在的 hash(`f26f3876c665a704d75a756f`)全替换掉。

### 2. 三个端口数据要联动

现在 3000、8082、8083 是 3 套完全独立的 mock。你创建的任务,在任鹏杰、张鼎功的页面里看不到。

方案:4 人共用一份 task json + node json 文件,或者跑一个共用 mock service(json-server 在 9000 端口)。会上一起定。

### 3. Nginx 反代

让 URL 看上去是一个系统:
- `http://fed-learn.local/scheduler/*` → 你的 3000
- `http://fed-learn.local/cloud/*` → 任鹏杰的 8082
- `http://fed-learn.local/edge/*` → 张鼎功的 8083

谁会 nginx 配置 30 分钟搞定。

### 4. 演示前必须切生产构建

现在跑的是 Vite dev 模式,首屏 8~13 秒。演示前 1 周必须切:

```
pnpm build
serve dist
```

切完首屏 1~2 秒。这一条不做演示当天必翻车。
